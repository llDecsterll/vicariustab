## 📖 О проекте

**Orbit** — передовая веб-система промышленного уровня для централизованного учета и инвентаризации IT-активов. Разработана для ИТ-отделов, системных администраторов и материально ответственных лиц. Позволяет объединить учет компьютеров, сетевой архитектуры, лицензий, расходных материалов на складе, а также закрепление за ними материально ответственных сотрудников.

Вся система полностью готова к работе в распределенном режиме: данные хранятся на центральном сервере СУБД, к которому пользователи могут безопасно подключаться **с любого компьютера, планшета или мобильного устройства через любой современный браузер**. Данные мгновенно обновляются, автоматически сохраняются и шифруются алгоритмом `AES-256`.

### Лицензирование и пробный период

- **Orbit поставляется без ключа активации** — для каждой установки автоматически генерируется **Уникальный Код Запроса Лицензии** (`REQ-XXXX-XXXX-XXXX-CHKS`).
- **30 дней ознакомительного периода** с первого запуска; по истечении срока доступ блокируется до активации.
- Ключ активации (`UTKIN-...`) выдаётся **только через отдельный сервер keyserver** (не входит в этот репозиторий) и привязан к MAC установки.
- **Product не может самостоятельно создать ключ** — в клиенте отсутствует генерация лицензий.
- **Копирование ключа с другой установки не работает** — проверка привязки к MAC, подписи активации и отпечатка оборудования.
- **Резервные копии и импорт БД** переносят только данные инвентаризации; лицензионные поля исключаются.

## ✨ Ключевые возможности

*   **Учет оборудования:** компьютеры, принтеры, коммутаторы, серверы, ноутбуки, комплектующие, история эксплуатации.
*   **Сетевое картографирование:** отслеживание структуры сетей, IP-адресов, патч-панелей и роутеров.
*   **Управление складом:** приход, списание, инвентаризационные описи, остатки картриджей и лицензий.
*   **Кадры и связки:** закрепление оргтехники за конкретными отделами и сотрудниками в реальном времени.
*   **Безопасность высшего уровня:** шифрование данных по стандарту `AES-256-CBC`, автоподключение к СУБД при сбоях и перезапуске хоста.

# 🛠️ Инструкция по установке системы Orbit на Ubuntu Server

В этом руководстве изложены все шаги от клонирования репозитория с GitHub до настройки СУБД (MySQL / PostgreSQL).

## Шаг 1. Подготовка и очистка сервера

Чтобы избежать конфликтов с путями и папками, выполняйте команды строго последовательно. Подключитесь к своей чистой машине Ubuntu Server по SSH и выполните очистку старых заготовок:

```bash
# Перейдите в домашнюю директорию root (или вашего пользователя)
cd ~

# Удалите старые конфликтные папки, чтобы установить всё "с чистого листа"
rm -rf Orbit
```

---

## Шаг 2. Скачивание проекта из GitHub
Склонируйте репозиторий проекта:

```bash
git clone https://github.com/llDecsterll/uvwstack.git
cd Orbit
```

*Обратите внимание: Все последующие команды должны запускаться строго внутри директории `~/Orbit`!*

---

## Шаг 3. Запуск системы Orbit (Выберите удобный способ)

### Вариант 1: Запуск через Docker & Docker Compose (Рекомендуется)

Данный способ автоматически развернет изолированную среду со встроенным веб-сервером и рантаймом.

#### 1. Установите Docker и Docker Compose на Ubuntu Server:
```bash
sudo apt update
sudo apt install -y docker.io docker-compose-v2
sudo usermod -aG docker $USER
```
*(Для активации прав пользователя Docker, выйдите из терминала SSH и зайдите снова).*

#### 2. Запуск контейнера приложения Orbit:
Запустите сборку и запуск приложения в фоновом автономном режиме:
```bash
docker compose build --no-cache
docker compose up -d
```
Система скомпилирует весь TypeScript и через несколько секунд станет доступна в браузере!

---

### Вариант 2: Локальная нативная установка на Ubuntu (без Docker, через PM2)

Используйте этот традиционный метод, если на сервере нет Docker или вы хотите запускать Orbit как нативную службу системы.

#### 1. Установите среду Node.js версии v20 LTS:
```bash
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs build-essential
```

#### 2. Установите зависимости и соберите проект:
```bash
cp .env.example .env
# Отредактируйте .env: задайте PORT и DB_ENCRYPTION_KEY
npm install
npm run build
```

#### 3. Настройка менеджера процессов PM2 для автозапуска 24/7:
```bash
sudo npm install -g pm2

# Запуск Node-сервера в фоновом режиме
PORT=8080 NODE_ENV=production pm2 start dist/server.cjs --name "orbit-system"

# Настройка автоматического старта службы при перезагрузке Ubuntu Server
pm2 startup systemd
```
*На экране появится длинная строчка с sudo, скопируйте её, вставьте в консоль и нажмите Enter.*

Затем сохраните текущую конфигурацию:
```bash
pm2 save
```

## Шаг 4. Установка и настройка Базы Данных на Ubuntu Server

Orbit поддерживает распределенное хранение данных в MySQL / MariaDB или PostgreSQL. СУБД гарантирует, что ваши данные **сохраняются после перезагрузки сервера, не теряются** и доступны всем пользователям, независимо от их браузера или компьютера.

Ниже приведены инструкции по развертыванию выбранной вами базы данных.

### Вариант А: Быстрая установка MySQL

#### 1. Установите пакеты и запустите службу:
```bash
sudo apt update
sudo apt install -y mysql-server
sudo systemctl start mysql
sudo systemctl enable mysql
```

#### 2. Настройте прослушивание портов со всех IP-адресов сервера:
```bash
sudo nano /etc/mysql/mysql.conf.d/mysqld.cnf
```
Найдите строку `bind-address` и измените её значение на:
```ini
bind-address = 0.0.0.0
```
Сохраните файл (`Ctrl+O`, затем `Enter`) и закройте его (`Ctrl+X`).

#### 3. Создайте базу данных и пользователя:
Войдите в интерактивную консоль MySQL:
```bash
sudo mysql
```
Выполните поочередно команды для настройки схемы и доступа:
```sql
-- Создание базы данных Orbit
CREATE DATABASE orbit_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

-- Создание администратора базы данных (Укажите свой пароль вместо StrongSecPassword@2026)
CREATE USER 'orbit_user'@'%' IDENTIFIED BY 'StrongSecPassword@2026';

-- Выдача прав пользователю
GRANT ALL PRIVILEGES ON orbit_db.* TO 'orbit_user'@'%';
FLUSH PRIVILEGES;
EXIT;
```

#### 4. Перезапустите службу и откройте порт в брандмауэре (UFW):
```bash
sudo systemctl restart mysql
sudo ufw allow 3306/tcp
sudo ufw reload
```

---

### Вариант Б: Быстрая установка PostgreSQL

#### 1. Установите СУБД PostgreSQL на ваш Linux:
```bash
sudo apt update
sudo apt install -y postgresql postgresql-contrib
sudo systemctl start postgresql
sudo systemctl enable postgresql
```

#### 2. Разрешите внешние подключения к PostgreSQL:
Откройте конфигурационный файл настроек сети:
```bash
sudo nano /etc/postgresql/*/main/postgresql.conf
```
Раскомментируйте параметр `listen_addresses` и установите его в значение прослушивания всех входящих интерфейсов:
```ini
listen_addresses = '*'
```
А затем откройте файл конфигурации клиентской авторизации:
```bash
sudo nano /etc/postgresql/*/main/pg_hba.conf
```
Добавьте в самый конец файла строчку разрешения для внешних хостов по паролю:
```text
host    all             all             0.0.0.0/0               scram-sha-256
```
Сохраните изменения в обоих файлах и закройте редакторы.

#### 3. Создайте схему и пользователя базы данных:
Войдите в терминал PostgreSQL от имени суперпользователя:
```bash
sudo -i -u postgres psql
```
Выполните команды:
```sql
CREATE USER orbit_user WITH PASSWORD 'StrongSecPassword@2026';
CREATE DATABASE orbit_db OWNER orbit_user;
\q
exit
```

#### 4. Перезагрузите службу и настройте порты брандмауэра:
```bash
sudo systemctl restart postgresql
sudo ufw allow 5432/tcp
sudo ufw reload
```

## Шаг 5. Сетевое подключение базы данных в Orbit

После запуска проекта и СУБД осталось связать их через интуитивный встроенный веб-интерфейс:

1. Откройте в вашем браузере страницу Orbit: `http://<IP_СЕРВЕРА>:8080`.
2. Авторизуйтесь с первоначальными данными:
   * **Логин:** `admin`
   * **Пароль:** `admin`
3. Откройте боковое меню и перейдите в **«Настройки»** -> Вкладку **«Параметры СУБД (MySQL / PostgreSQL)»**.
4. Заполните сетевые реквизиты:
   * **Тип:** Выберите установленную вами СУБД (`MySQL` или `PostgreSQL`).
   * **Хост / IP:** Укажите IP-адрес вашего Ubuntu-сервера. *(Если приложение крутится в Docker-контейнере, а СУБД расположена на самом сервере Ubuntu, укажите сетевой шлюз докера: `172.17.0.1`)*.
   * **Порт:** `3306` (если MySQL) или `5432` (если Postgres).
   * **База Данных:** `orbit_db`.
   * **Пользователь:** `orbit_user`.
   * **Пароль:** Тот пароль, который вы задали ранее при создании пользователя (например, `StrongSecPassword@2026`).
5. Нажмите кнопку **«Проверить соединение»**. Появится зеленая надпись **«Подключение успешно установлено!»**.
6. Нажмите кнопку **«Применить СУБД и мигрировать»**. Система автоматически:
   * Зашифрует параметры доступа методом `AES-256-CBC` для вашей безопасности;
   * Автоматически развернет и создаст всю структуру необходимых таблиц в базе;
   * Перенесет все ранее добавленные вами локальные записи ТМЦ из буфера в постоянную СУБД без потери информации;
   * Настроит систему на **автоматическое бесшовное подключение к СУБД** в случае перезапусков, сбоев или технических перезагрузок сервера!

## 📂 Структура проекта

```text
Orbit/
│
├── src/                      # React UI + утилиты (license, i18n)
├── server.ts                 # Express API, шифрование db.json, SQL-миграции
├── Dockerfile                # Двухэтапная сборка Node.js
├── docker-compose.yml        # Запуск в Docker
├── nginx.conf                # Проксирование (при необходимости)
├── package.json              # Зависимости и скрипты
├── .env.example              # PORT, DB_ENCRYPTION_KEY
├── README.md                 # Установка и администрирование
└── DOCKER.md                 # Docker-инфраструктура
```

### Переменные окружения

| Переменная | Описание |
|---|---|
| `PORT` | Порт HTTP-сервера (по умолчанию `3000`, для продакшена часто `8080`) |
| `DB_ENCRYPTION_KEY` | Секрет AES-256 для `db.json` и зашифрованных настроек СУБД |
| `NODE_ENV` | `production` для статической раздачи из `dist/` |
